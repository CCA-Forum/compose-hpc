Program Contract Annotation(s) Purpose
Routine/Method Inlined
helloworld.cc FINAL, INIT, REQUIRE, STATS An example containing only routine-level annotations and an enforcement configuration file resulting in enforcement statistics output.
helloworld-v2.cc REQUIRE FINAL, INIT, STATS An example containing mostly in-lined annotations, which means much of the enforcement instrumentation can only be added through the VisitContractsInstrumenter driver. Unfortunately, this is also a case where the in-lined finalization annotation appearing immediately before the return does not get translated (at least with ROSE 0.9.5a-17286).
helloworld-v3.cc REQUIRE FINAL, INIT An example containing routine and in-lined annotations placed such that the finalization annotation does get translated. An enforcement configuration file is also provided and enforcement statistics output through the FINAL instrumentation.
knapsack.c ENSURE, FINAL, INIT, REQUIRE ASSERT An example with simple, labeled routine contracts and in-lined assertions translated with the VisitContractsInstrumenter driver.
Knapsack-v2.cpp ENSURE, FINAL, INIT, INVARIANT, REQUIRE ASSERT An example with labeled routine contracts and in-lined assertions but an advanced, unsupported assertion expression that does not get translated by either transformation driver.
unlabeledknapsack.c ENSURE, FINAL, INIT, REQUIRE ASSERT An example with unlabeled routine contract and in-lined assertion annotations.
UnlabeledKnapsack-v2.cpp ENSURE, FINAL, INIT, INVARIANT, REQUIRE ASSERT An example with unlabeled routine contract and in-lined assertion annotations but an advanced, unsupported assertion expression that does not get translated by either transformation driver.
WeightsList.cpp ENSURE, FINAL, INIT, INVARIANT, REQUIRE An example with labeled routine contracts whose purpose is to trigger a class invariant violation when contracts are enforced.
